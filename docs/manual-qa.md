# 手工验收清单（Manual QA）

按场景逐步验证 MVP 核心路径。每项含操作步骤、预期结果、常见失败原因。

> 前置：已 `pnpm install` 并创建 Python venv（见 README）。

---

## 场景 1：启动 Web

**步骤**：
1. 终端 A：`pnpm dev:api`
2. 终端 B：`pnpm dev:web`
3. 浏览器打开 http://127.0.0.1:5173

**预期**：
- 页面加载，顶部连接徽章显示「已连接 v0.8.0」（绿色）
- 画布区显示「左侧上传扫描稿图片开始编辑」

**常见失败**：
- 徽章显示「未连接」→ 后端未启动，检查 `pnpm dev:api` 是否运行在 8001
- 页面空白 → vite 未就绪，等几秒刷新

---

## 场景 2：启动后端

**步骤**：`pnpm dev:api`

**预期**：终端显示 `Uvicorn running on http://127.0.0.1:8001`

**常见失败**：
- 端口占用 → 关闭占用进程或改端口
- ModuleNotFoundError → 检查 venv 是否装了 requirements.txt

---

## 场景 3：加载 demo 工程

**步骤**：
1. 点顶部「加载」按钮
2. 选择 `storage/samples/demo-project.json`

**预期**：
- 画布显示第 1 页（笔记本背景 + 2 段文字）
- 页面列表显示 2 页
- 手写档案显示「演示手写档案（合成）」+ 5 个 glyph
- naturalness 开关已开启

**常见失败**：
- 加载失败 → JSON 损坏，重新运行 `scripts/gen_demo_project.py`

---

## 场景 4：创建多页项目

**步骤**：
1. 点页面面板「+ 新页」
2. 点「📥 多图」→ 选多张图片

**预期**：
- 新页出现在列表，页码更新（1/N）
- 多图按文件名排序生成多页

**常见失败**：
- 图片顺序乱 → 多图按文件名排序，建议命名 01.png 02.png

---

## 场景 5：清除区域

**步骤**：
1. 上传/加载有背景图的页
2. 勾选「框选模式」
3. 在背景上拖拽选区
4. 点「清除字迹」
5. 点「↶ 撤销清除」

**预期**：
- 选区可调整大小（拖角）
- 清除时按钮显示「处理中…」
- 清除后该区域内容被填充
- 撤销恢复原背景

**常见失败**：
- 清除失败报错 → 后端未连接，检查 `/clean-region`
- 切换页面后清除串页 → 不应发生，清除历史按页隔离

---

## 场景 6：手写档案 + glyph 标注

**步骤**：
1. 展开「手写档案」→ 新建档案
2. 导入样本图（如 demo-notes.png）
3. 点「切割字形」→ 「自动检测字形区域」
4. 选中候选框 → 输入字符 → 保存（Enter）
5. 用 ← → 切换候选框，连续标注多个

**预期**：
- 自动检测出候选框（绿/黄/红颜色提示质量）
- 保存后候选框消失，自动跳下一个
- glyph 列表显示已保存字形

**常见失败**：
- 检测无结果 → 样本图对比度低或字太小，调整书写
- 保存失败 → 后端 `/segment-glyph` 未响应

---

## 场景 7：OCR unavailable 手动标注

**步骤**：在切割器中（未装 OCR）

**预期**：
- 「OCR 辅助识别」按钮禁用
- 显示「OCR 未启用，可继续手动标注」
- 手动标注、批量标注正常

---

## 场景 8：mock OCR 辅助标注

**步骤**：
1. 后端用 `HW_OCR_MOCK=1 pnpm dev:api` 启动
2. 切割器中点「OCR 辅助识别」
3. 点「应用高置信度建议」

**预期**：
- 候选框显示 OCR 建议百分比
- 高置信度（绿）可一键应用
- 低置信度（灰）字符为空

---

## 场景 9：handwritingGlyph 文本

**步骤**：
1. 添加文本框，输入「我的笔」
2. StylePanel 切换「手写素材」模式
3. 选择手写档案

**预期**：
- 有 glyph 的字显示为字形图片
- 缺字显示「缺少字形：X」并 fallback 字体
- 显示覆盖字形数

---

## 场景 10：导出

**步骤**：
1. 导出 PNG（当前页）
2. 导出 页 PDF（当前页）
3. 导出 全 PDF（全部页）
4. 导出 PNG ZIP（全部页）
5. 切换倍率 1x/2x

**预期**：
- 文件名含时间戳和页码
- PDF/ZIP 非活动页也含 glyph
- 导出过程显示进度
- 缺字时弹窗提示（可选继续）

**常见失败**：
- 导出空白 → 检查 stage 是否挂载
- 大图失败 → 超过 canvas 限制（16384px）

---

## 场景 11：保存/加载项目

**步骤**：
1. 修改内容（标题栏出现红点 ●）
2. 点「保存」→ 下载 JSON
3. 刷新页面
4. 点「加载」→ 选刚才 JSON

**预期**：
- 修改后红点出现，刷新前浏览器警告
- 保存后红点消失
- 加载后多页、手写档案、glyph 全部恢复

---

## 场景 12：撤销/重做

**步骤**：
1. 添加文本 → Ctrl+Z → Ctrl+Y
2. 删除文本 → Ctrl+Z
3. 切换页面后再撤销

**预期**：
- Ctrl+Z 撤销，Ctrl+Y 重做
- 切换页面后撤销栈不串页

---

## 场景 13：桌面 dev

**步骤**：
1. `pnpm dev:web`（终端 A）
2. `pnpm dev:desktop`（终端 B）

**预期**：
- Electron 窗口打开，加载 vite 内容
- 若后端未启动，弹窗提示如何启动

---

## 场景 14：桌面 prod

**步骤**：
1. `pnpm build:desktop`
2. `pnpm --filter @hw-layout/desktop start`

**预期**：
- Electron 加载 web/dist 构建产物
- 无 vite 依赖

**常见失败**：
- 白屏 → 检查 apps/web/dist 是否存在（先 `pnpm build:web`）

---

## 场景 15：空状态

**步骤**：分别测试无图、无档案、disconnected 状态

**预期**：
- 无图：画布区显示上传引导
- 无档案：手写档案面板显示「还没有手写档案」
- disconnected：连接徽章红色，功能受限提示
