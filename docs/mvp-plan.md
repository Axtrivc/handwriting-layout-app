# MVP 计划

## 当前阶段：MVP 骨架（第一轮）

### 目标

搭建可运行的本地桌面应用骨架，打通「前端编辑 → 后端清理 → 导出」最小闭环。

### MVP 功能清单

1. **Electron 桌面壳**：能启动并加载 React 前端
2. **React 前端页面**：
   - 上传区：选择本地图片
   - 扫描稿画布（Konva.js）：预览上传图片
   - 文字输入区：添加文本框
   - 样式控制区：位置、字号、字距、行距、颜色、透明度、轻微旋转
   - 导出按钮：前端 canvas 导出 PNG
3. **画布交互**：
   - 添加文本框，可拖动调整位置
   - 选中后可在样式控制区调整各项参数
4. **手写自然化开关**：
   - 对每个文字对象加入轻微位置抖动、旋转抖动、透明度变化
   - 参数温和（位置 ±2px，旋转 ±1.5°，透明度 ±5%）
5. **FastAPI 后端**：
   - `GET /health`：健康检查
   - `POST /clean-region`：接收图片 + 矩形区域，OpenCV inpaint 基础版
   - `POST /export`：预留接口（MVP 用前端导出）

### MVP 不做的事

- 自动识别整页字迹
- 手写字体生成 / 笔迹风格迁移
- PDF 导出（预留接口，后续阶段）
- 多页工程管理
- 云同步

## 后续阶段

### 第二阶段：清理能力增强

- 多种 inpaint 算法可选（Telea / NS / 背景填充）
- 区域批量框选与撤销
- 去污、二值化、对比度调整等扫描稿修复工具

### 第三阶段：排版工程化

- 多页工程
- 文本流式排版（自动换行、分栏）
- 模板系统（保存/加载版式）
- 图层管理

### 第四阶段：自动辅助切字 + 渲染补强（已完成）

- 自动辅助切字：连通域检测字形候选区域 + 行聚类 + 阅读顺序排序（`/detect-glyph-candidates`）
- 前端批量标注：按阅读顺序匹配字符，一次性保存多个字形
- 候选框可删除 / 调整 / 新增
- glyph 质量检查（good/warning/poor 提示，不阻止保存）
- handwritingGlyph 对齐修复（left/center/right + 多行）
- 渲染自然度增强：scale jitter / baseline jitter / blur / opacity（seed 稳定）
- 缺字提示 + 覆盖率统计（总 glyph / 覆盖字符 / 多 variant / 缺字列表）
- 项目 JSON 向后兼容

### 第五阶段：手写风格（合规范围内）

- 采集**用户本人**手写样本
- 生成**本人风格**的字形变体（仅本人使用）
- 个人手写排版优化

### 第五轮：多页工程与 PDF 导出（已完成）

- 多页数据结构：CanvasPage[] + activePageId，旧单页项目自动迁移为 pages[0]
- PagePanel：新建/删除(确认)/复制/重命名/上移下移/切换/缩略图/页码
- 多图导入：一次选多张图，按文件名排序自动建多页
- 页面级隔离：每页独立背景/文本/清除历史，切换不串页
- handwritingProfiles 仍在 project 级共享
- PDF 导出：jsPDF，当前页 PDF + 全部页 PDF，按原图比例，含背景/文本/glyph/naturalness
- PNG 导出升级：当前页，文件名带页码（-page-001）
- 项目 JSON 多页 round-trip，旧项目兼容

### 第六阶段：导出与集成

- PDF 导出（矢量 + 光学）
- 工程文件存档
- 本地历史记录
