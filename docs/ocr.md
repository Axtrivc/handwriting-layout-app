# OCR 辅助标注说明

## 重要：OCR 是辅助，不是自动确认

OCR 仅作为**辅助建议**，帮助用户更快标注字形。它**不自动替用户确认**字形标注：
- 用户必须检查、修改、确认后再保存
- 低置信度建议不会自动填入
- OCR 识别失败不阻塞手动标注

OCR 识别准确率受书写质量影响，**不保证准确**。

## OCR 是否必需？

**OCR 是可选依赖**，不安装也能正常使用应用：
- 未安装 OCR 时，UI 显示「OCR 未启用，可继续手动标注」
- 自动检测候选框、手动框选、批量标注等功能**不受影响**
- 只有「OCR 辅助识别」按钮不可用

## 如何启用 OCR

### 方案一：RapidOCR（推荐，轻量）

```bash
cd services/api
.venv/Scripts/python.exe -m pip install rapidocr-onnxruntime
```

安装后重启后端（`pnpm dev:api`），应用会自动检测到 RapidOCR 并启用。

### 方案二：Mock provider（仅供测试）

```bash
# 设置环境变量启用 mock（返回假候选，用于测试 OCR 流程）
HW_OCR_MOCK=1 pnpm dev:api
```

Mock 不做真实识别，仅供开发测试。

### 关于 EasyOCR

EasyOCR 较重（依赖 torch），当前仅注册占位。如需启用，参考 `services/vision/vision/ocr.py` 中的 `_try_import_easyocr` 补全调用逻辑。

## OCR 未启用时如何继续

1. 用「自动检测字形区域」找出候选框
2. 在候选框列表中逐个输入字符（← → 切换、Enter 保存）
3. 或用「批量标注」输入一串字符按阅读顺序匹配

完全不需要 OCR 也能完成全部标注工作，OCR 只是加速器。

## 提升 OCR 准确率的书写建议

为了让 OCR 辅助识别更准，采集样本时注意：

- **字间距更大**：每个字之间留足空隙，便于切割
- **每个字尽量完整**：笔画清晰，不要潦草到无法辨认
- **不要连笔过重**：连笔会导致切割粘连、识别困难
- **深色笔 + 浅色纸**：黑/蓝笔 + 白纸，对比度高
- **拍照/扫描保持清晰**：对焦准确、分辨率足够
- **避免阴影和倾斜**：光线均匀，横向书写

## 置信度分级

OCR 建议按置信度分三级显示：

| 级别 | 置信度 | 行为 |
|------|--------|------|
| high（绿） | ≥ 0.85 | 可「一键应用」批量填入 |
| medium（黄） | 0.60 ~ 0.85 | 显示建议，但**不自动应用**，需手动确认 |
| low（灰） | < 0.60 | 字符置空，仅显示置信度提示 |

**低置信度建议默认为空**，避免误导。用户应优先检查 medium/low 的候选框。

## 合规说明

OCR 仅辅助用户标注**自己**的手写样本，用于个人笔记排版。
不识别他人笔迹，不用于签名、证件、合同、票据、考试/作业冒充。
详见 [product-boundary.md](product-boundary.md)。
